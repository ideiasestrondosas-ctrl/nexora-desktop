## 2. Ficheiro 1 — `src/pages/AssetDetailPage.tsx`

### 2.1 Bug: overlay do player mostra sempre o caminho original

**Localização:** linhas 272–275 — o `<div>` de overlay dentro do container do player mostra sempre `asset.path`.

**Fix:** Substituir `asset.path` pelo mesmo cálculo condicional já usado nas linhas 281–285:

```
heroView === 'out' && (asset.output_path ?? jobs[0]?.output_path)
  ? (asset.output_path ?? jobs[0]?.output_path)
  : asset.path
```

### 2.2 Bug: botão "Processar Novamente" não executa

**Causa:** `invoke('submit_job', { assetId, ... })` usa camelCase — o backend Tauri espera `asset_id` (snake_case).

**Fix:**

- Corrigir para `invoke('submit_job', { asset_id: assetId, profile, priority: 0 })`.
- Adicionar state `reprocessMenuOpen: boolean` e lista de perfis carregada via `invoke<string[]>('list_profiles')` no mount.
- O botão abre um dropdown inline com os perfis disponíveis; selecionar um chama `handleReprocess(profile)`.
- Após submissão com sucesso: toast de confirmação + `fetchData()`.

### 2.3 Bug: botão "Abrir no Explorador" só abre o original

**Fix:** Substituir o botão único por dois botões na action bar:

- **"Ver Original"** → `revealItemInDir(asset.path)`
- **"Ver Processado"** → `revealItemInDir(output_path)` — desativado (`opacity-40`, `cursor-not-allowed`) se não existir `output_path`

### 2.4 MediaInfo — mostrar caminho do ficheiro original

**Situação:** Na secção "Metadados Técnicos", o lado "Original" não mostra o caminho. O lado "Processado" já mostra.

**Fix:** Adicionar idêntico componente de path (ícone `FolderOpen` + caminho `asset.path`) no lado "Original", com estilo `bg-brand/5 border-brand/20 text-brand/70`.

### 2.5 Tabs horizontais no MediaInfoPanel (sub-secções)

**Situação:** As sub-secções (General, Vídeo, Áudio, Tags, SHA-256) são painéis colapsáveis verticais dentro do `MediaInfoPanel`.

**Design:**

```
┌─────────────────────────────────────────────────────┐
│ [ Geral ] [ Vídeo ] [ Áudio ] [ Etiquetas ] [ SHA ] │
├─────────────────────────────────────────────────────┤
│  conteúdo da tab activa (rows existentes)           │
└─────────────────────────────────────────────────────┘
```

- Estado local `activeTab: 'general' | 'video' | 'audio' | 'tags' | 'sha256'`, começa em `'general'`.
- Tabs desativadas se não houver dados (SHA-256 sem `info.sha256`, Áudio sem `audioStreams`, etc.).
- O componente `Section` e `SectionProps` são removidos.
- O botão "Copiar tudo" e `generateTextReport()` mantêm-se inalterados no topo.
- Múltiplos streams de vídeo/áudio renderizados sequencialmente dentro da tab correspondente.

### 2.6 Comparação técnica — caminhos + tamanho de ficheiro

**Adições à coluna "Original"** (antes das rows técnicas existentes):

- Linha "Ficheiro": `asset.filename`
- Linha "Caminho": `asset.path` (truncado, com `title` para tooltip)
- Linha "Tamanho": `formatBytes(asset.size_bytes)`

**Adições à coluna "Processado"** (mesmas três linhas):

- Linha "Ficheiro": nome extraído de `output_path.split(/[/\\]/).pop()`
- Linha "Caminho": `asset.output_path`
- Linha "Tamanho": `formatBytes(asset.output_metadata?.format?.size)` — usa o campo `size` do ffprobe que já existe no JSON; se não disponível mostra `—`.

### 2.7 Novo botão: Download do ficheiro processado

**Posição:** Action bar sticky, entre "Processar Novamente" e "Apagar".

**Comportamento:**

1. Verificar se `output_path` existe; se não, `logActivity('download_processed', 'attempt')` + toast de aviso.
2. Chamar `save({ defaultPath: filename })` do `@tauri-apps/plugin-dialog`.
3. Se o utilizador confirmar, `copyFile(output_path, destino)` do `@tauri-apps/plugin-fs`.
4. Toast de sucesso com o caminho de destino.

**Ícone:** `Download` do lucide-react.

## 4. Ficheiro 3 — `src/pages/QueuePage.tsx`

### 4.1 Bug: ícone "abrir asset" não navega para o detalhe

**Localização:** Tabela de concluídos — botão `ExternalLink` chama `openPath(job.output_path!)`.

**Fix:** Substituir por `onSelectAsset?.(job.asset_id)`. O ícone mantém-se.

### 4.2 Ícone "processar" oferece opções de perfil

**Situação atual:** Botão `Repeat` chama `handleRetry(job.id)` sem opções.

**Design:**

- Adicionar state `reprocessPopover: string | null` (guarda o `job.id` com o popover aberto).
- Adicionar state `availableProfiles: string[]` carregado no mount via `invoke('list_profiles')`.
- Clicar no `Repeat` abre um popover inline com:
  - Opção "Mesmo perfil (`job.profile`)" → chama `handleRetry(job.id)`
  - Separador
  - Lista dos outros perfis → chama `handleReprocessWithProfile(job.asset_id, profile)`
- `handleReprocessWithProfile`: `invoke('submit_job', { asset_id, profile, priority: 0 })` + toast + `fetchData()`.
- Fechar popover ao clicar fora (listener `mousedown` no `document`).

## 7. Novo ficheiro — `src/lib/activityLog.ts`

### 7.1 API pública

```ts
type ActivityType = 'click' | 'execute' | 'attempt' | 'navigate';

function logActivity(label: string, type: ActivityType, details?: string): void;
```

### 7.2 Implementação

- Escreve num Zustand store `useActivityLogStore` (in-memory, máx. 500 entradas, FIFO).
- Formata a mensagem: `[ACTIVITY] [timestamp] type | label | details`.
- Chama `invoke('log_activity', { message })` para persistir no log Tauri — se o comando não existir, usa `console.info` como fallback silencioso.

### 7.3 Integração nos componentes

Todos os handlers de botões em `AssetDetailPage`, `QueuePage`, `LibraryPage` e `DashboardPage` chamam `logActivity` com:

- Tipo `'execute'` para ações que avançam com sucesso.
- Tipo `'attempt'` para ações bloqueadas por falta de dados (ex: download sem output_path).
- Tipo `'click'` para navegações e toggles.

Exemplos:

```
[ACTIVITY] click | "Aba Metadados Técnicos" | asset_id=abc123
[ACTIVITY] execute | "Processar Novamente" | asset_id=abc123 profile=broadcast-hd
[ACTIVITY] attempt | "Download Processado" | sem output_path disponível
[ACTIVITY] navigate | "Fila → Detalhe Asset" | asset_id=abc123
```

### 7.4 Visibilidade

As entradas `[ACTIVITY]` aparecem no `LogsPage` existente automaticamente (o LogViewer já lê do log Tauri). Não são necessárias alterações ao LogViewer.
