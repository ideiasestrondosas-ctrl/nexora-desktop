## 2. Instruções de Implementação e Regras de Ouro

### [CONFIGURAÇÃO DE AMBIENTE E WORKSPACE]

1.  **Validação de Diretório:** Confirme se `C:\Dev\Nexora Media Processing\nexora-desktop` está pronto para uso. Avalie como utilizar o projeto pai (na pasta acima) como fonte de lógica/recursos sem duplicar código desnecessariamente. 2.  **Escopo do Workspace:** Para você (Antigravity), considere o workspace na raiz `C:\Dev\Nexora Media Processing` para ter visão global, mas **RESTRINJA TODA ESCRITA** exclusivamente à subpasta `nexora-desktop`.
3.  **Permissões:** O projeto base (`C:\Dev\Nexora Media Processing`) deve ser tratado como **APENAS LEITURA**. Qualquer alteração necessária no "core" deve ser discutida como uma proposta de adaptação, nunca alterada diretamente sem aviso.

### [ADAPTAÇÃO DO GUIA PARA LEIGOS]

4.  **Guia Passo a Passo:** Adapte o arquivo `nexora-desktop-guia-execucao.md` para ser o meu **Manual de Navegação**. Como sou leigo, o guia deve ser "verboso" na explicação do _porquê_ de cada comando e o que esperar de resultado. Inclua prints (se possível via ferramentas de browser) ou descrições claras de sucesso.

### [GOVERNANÇA E REGRAS DA IA (Karpathy/Graphify/Skills)]

5.  **Ativação de Regras:** Configure as regras de desenvolvimento baseadas nos padrões `Graphify` (mapeamento de dependências) e `Karpathy` (código limpo, legível e denso em lógica útil).

7.  **Uso de Skills:** Utilize ativamente suas `skills` (filesystem-context, webapp-testing, ui-ux-pro-max) para garantir que o app desktop seja premium.

### [COEXISTÊNCIA: ANTIGRAVITY + CLAUDE]

8.  **Proteção de Configurações:** **NÃO ALTERE** arquivos de configuração do Claude (`.cursor`, `.claudespec`, etc.) ou definições que possam conflitar com a outra IA. Mantenha um "Tratado de Não Agressão" entre as ferramentas.
